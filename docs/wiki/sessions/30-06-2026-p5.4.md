# Summary

Implemented the write plane (P5.4): the agent's `Promote` and `Rollback` RPCs now
run a real production deploy and server-stream the build/deploy log as `LogLine`.
Extracted the deploy orchestration out of `cli` into a new `internal/deployflow`
package so the CLI and the agent drive the same promotion state machine. `Deploy`
(staging) stays `Unimplemented` — no release-lifecycle hook this slice.

# Files Touched

- `internal/deployflow/deployflow.go` — new. `Service.ToProduction(ctx, r, sink)`:
  record pending deployment → run deploy (streaming to `LogSink`) → on failure mark
  failed; on success `WithTx{archivePrevious; →production; mark succeeded}`. Owns the
  `Deployer`, `TxRunner`, `LogSink` ports + a `WriterSink` adapter for the CLI.
- `internal/deploy/deploy.go` — `Run`/`Script.Run` gained a `LogSink` param; stream
  stdout+stderr line by line via `StdoutPipe`/`StderrPipe` + `bufio.Scanner` drained
  by a `WaitGroup` before `cmd.Wait`.
- `internal/cli/cli.go` — `App.deploy` delegates to `deployflow.Service`; `Deployer`/
  `TxRunner` interfaces moved to deployflow. Behavior unchanged.
- `internal/agent/server.go` — `Promote`/`Rollback` implemented: validate (service,
  prod env, hex commit), single-flight `sync.Mutex.TryLock`, resolve release by
  commit, status guard (staging / archived), stream `ToProduction` under a detached
  context (`context.WithoutCancel` + 15m cap). `New` gained a `*deployflow.Service`.
- `cmd/tsugi/main.go` — `serve` builds the `deployflow.Service` and wires it in.
- Tests: `internal/agent/write_test.go` (new) covers validation, status guards,
  NotFound, success + failure; `cli_test` `fakeDeployer.Run` gained the sink param.

# Previous Behavior

`Promote`/`Rollback` returned `codes.Unimplemented`. The deploy orchestration lived
only in `cli.App.deploy`, writing to the CLI's `io.Writer`; `deploy.Run` inherited
the process stdout/stderr.

# New Behavior

Yagura (localhost) drives a production deploy over gRPC and relays the streamed log
to the browser. The orchestration is shared, so CLI and agent can't drift on the
single-active-production / archive-previous invariants.

# Reason For Change

P5.4 exit (Yagura PLAN §13): click Deploy → logs stream live → release row updates.

# Risks

- **Detached deploy context**: a dropped gRPC stream no longer aborts a running
  deploy (avoids half-deploys on a flaky viewer); a wedged deploy runs to the 15m cap.
- **Single-flight lock**: a second concurrent deploy is rejected (`FailedPrecondition`),
  not queued — a git checkout + compose race would corrupt the target.
- The write RPCs shell `docker`/`git` on the host; the `tsugi` account must join the
  docker group (noted in the P5.2 systemd unit risk).

# Notes

`go build`/`go vet`/`go test -race ./...` green. No `.proto` change — the contract
already carried the streaming RPCs from P5.1. Validation mirrors `deploy.sh`'s `--ref`
charset guard (defense in depth).
